介绍 rocketmq 消息跟踪功能

前言

消息跟踪，可以跟踪消息由哪个 client 产生，保存在哪个 broker ， 是否被消费以及由哪个 client 消费等一系统消息的属性。 这个功能可以方便消息的调试的故障排查。

consumer 打开消息跟踪的方法

消息跟踪会用消息的形式保存到 RMQ_SYS_TRACE_TOPIC 这个 topic, 可以用 console 查看，也可以自行解析 topic 里的消息查看。
