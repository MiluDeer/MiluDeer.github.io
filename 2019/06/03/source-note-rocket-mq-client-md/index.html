【阅读笔记】rocketmq 客户端

发表于 2019-06-03 | 更新于 2019-06-13

简介

rocketmq 客户端在 rocketmq-client 工程实现，它定义了两类主要的客户端 producer, consumer。在同一个应用中，所有的 client 都由 MQClientManager 进行管理，MQClientManager 采用单例模式，它是一个 MQClientInstance 工厂，生成进保存 MQClientInstance 对象。而一个 MQClientInstance 对象就是一个 client，它包含 client 对外通信所需要的成员。

如图， MQClientAPIImpl 包含有成员 remotingClient，所以它可以同其它 rocketmq 组件进行通信，所有 client 对外的请求都通过 MQClientAPIImpl 封装起来。例如 MQClientAPIImpl#createTopic 方法向 broker 发送 UPDATE_AND_CREATE_TOPIC 命令创建 topic，这个方法所要做的事就是构建一个 RemotingCommand 并通过 remotingClient 发送出去。

MQClientInstance 除了包含 MQClientAPIImpl，还包含如下成员：

  • clientId：用于标识不同的 client
  • producerTable: 按 producer group 保存了 producer 的信息
  • consumerTable：按 consumer group 保存了 consumer 的信息
  • adminExtTable：
  • topicRouteTable： 保存各个 topic 的信息
  • brokerAddrTable：保存 broker 地址
  • brokerVersionTable： 记录 broker 的数据版本号
    …

MQClientManager 创建并通过 ConcurrentMap 保存了多个 MQClientInstance 实例， 在需要使用 MQClientInstance 时，先通过 MQClientManager 获取。

前言

producer 负责产生消息发到 broker。有两种 producer： DefaultMQProducer 负责普通的消息生产；TransactionMQProducer 继承 DefaultMQProducer 负责消息事务。

配置

Clinet 公共配置有

  • namesrvAddr namesrv 地址，默认会通过环境变量 NAMESRV_ADDR 或 系统变量 rocketmq.namesrv.addr 读取
  • clientIP clinet 的 ip 地址
  • instanceName 实际名称，会从系统变量 rocketmq.client.name 读取，否则就是进程 ID
  • clientCallbackExecutorThreads 执行 callback 的线程数
  • pollNameServerInterval 从 namesrv 拉 topic 信息的时间间隔，默认 1000 * 30 ms
  • heartbeatBrokerInterval 到 broker 的心跳时间，默认 1000 * 30 ms
  • persistConsumerOffsetInterval consumer 用的，消费位置的持久化时间间隔，默认 1000 * 5 ms
  • vipChannelEnabled 是否开启 vip 通道，可以用系统变量 com.rocketmq.sendMessageWithVIPChannel 配置

producer 配置

  • producerGroup producer 组名，使用消息事务时有用
  • createTopicKey 自动创建 topic 时用
  • defaultTopicQueueNums 自动创建 topic 时，默认的队列数，读写都是这个数
  • sendMsgTimeout 发送消息的超时时间
  • compressMsgBodyOverHowmuch 消息多大时采用压缩，默认 4 K
  • retryTimesWhenSendFailed 发送失败重试次数
  • retryTimesWhenSendAsyncFailed 异步发送失败重试次数
  • retryAnotherBrokerWhenNotStoreOK 是否从另一个 broker 重试
  • maxMessageSize 消息最大长度， 4 M
